Co to Popover API?

Popover API to nowoczesny sposób tworzenia „wyskakujących elementów” (tooltipów, menu, podpowiedzi), które pojawiają się nad stroną, ale nie blokują interakcji tak jak modal.

W przeciwieństwie do <dialog>, popover nie blokuje reszty strony i działa bardziej jak lekki overlay.

Jak stworzyć popover?

Wystarczy użyć atrybutu popover na dowolnym elemencie HTML.
Domyślnie element jest ukryty i pojawia się dopiero po wywołaniu.

Sposób 1 — showPopover() i hidePopover()

To jest popover otwarty metodą showPopover().

const popover = document.getElementById("popover");
const openBtn = document.getElementById("open");
const closeBtn = document.getElementById("close");

openBtn.onclick = () => {
  popover.showPopover();
}

closeBtn.onclick = () => {
  popover.hidePopover();
}

Sposób 2 — Invoker Commands API

Tak jak w modalu, możemy używać deklaratywnych komend bez JavaScript.

Popover otwarty bez JavaScript.

<button popovertarget="popover"
  popovertargetaction="show">
  Otwórz
</button>

<div id="popover" popover>
  <button popovertarget="popover"
    popovertargetaction="hide">
    Zamknij
  </button>
</div>

Różnica między Popover a Modal

Popover

✔ Nie blokuje strony
✔ Może być lekko interaktywny (tooltip, menu)
✔ Nie wymusza zamknięcia

Modal (<dialog>)

✔ Blokuje resztę strony
✔ Wymusza interakcję (np. zamknięcie)
✔ Używany do ważnych akcji (formularze, potwierdzenia)

Dodatkowa funkcja

Popover automatycznie zamyka się po kliknięciu poza elementem — nie trzeba pisać dodatkowego JavaScriptu jak w dialogu.

Rodzaje popoverów

Popover API może działać w kilku trybach — w zależności od tego, jak bardzo chcemy kontrolować jego zachowanie.

1. popover="auto"

Domyślny tryb. Popover automatycznie zamyka się, gdy klikniesz poza nim lub otworzysz inny popover.

Najczęściej używany do menu i tooltipów.

<div popover="auto">
  Treść popovera
</div>

2. popover="manual"

W tym trybie masz pełną kontrolę nad otwieraniem i zamykaniem.
Nie zamknie się sam po kliknięciu poza elementem.

Używane np. w niestandardowych panelach UI.

<div popover="manual">
  Treść popovera
</div>

3. popover="hint"

Tryb „podpowiedzi” (hint) używany do lekkich UI jak tooltipy.

• znika szybciej niż auto
• nie wymaga pełnej interakcji
• idealny do hover / focus / podpowiedzi

<div popover="hint">
  Podpowiedź UI
</div>

4. Light-dismiss

Mechanizm zamykania po kliknięciu poza popover.
Działa automatycznie tylko w trybie auto.

Pozycjonowanie popoverów

Popover API nie narzuca pozycji na ekranie. Każdy popover trafia do specjalnej warstwy top-layer, a jego pozycja zależy od CSS.

1. Warstwa top-layer

Popover nie działa jak zwykły element DOM. Jest renderowany nad całą stroną, niezależnie od z-index.

2. Pozycjonowanie klasyczne (CSS)

Możemy ustawić go ręcznie jak overlay:

[popover] {
  position: fixed;
  top: 120px;
  left: 120px;
}

To podejście działa, ale jest mało elastyczne — nie reaguje na przycisk wywołujący.

3. Anchor Positioning (nowoczesne UI)

Najnowszy i najlepszy sposób pozycjonowania popoverów. Pozwala „przypiąć” popover do elementu wyzwalającego.

button { anchor-name: --btn; }

#popover {
  position-anchor: --btn;
  top: anchor(bottom);
  left: anchor(center);
}

4. Dlaczego to ważne?

Dzięki temu popovery mogą działać jak:
• tooltipy przy przyciskach
• dropdowny
• menu kontekstowe
• podpowiedzi UI
